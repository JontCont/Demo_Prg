Git與Github？

GIT 是一種分散式版本控制的軟體，主要對 GITHUB 作為媒介上傳至GITHUB ，最初目的是為更好地管理 Linux kernel 開發而設計，其具備優秀的 merge tracing 合併程式碼的能力。
GITHUB 支援 git 程式碼存取和遠端托管的平台服務，許多專案是開放原始碼，使用 Github 進行程式碼的管理。

Git 基本觀念

在Git工作流程中，使用先從儲存庫下載某一版本到工作目錄，檔案修改完後，把檔案快照新增到暫存區域，執行提交後檔案就永久存到儲存庫內。

Git 可以分為 Local 和 Remote 兩個環境，由於 Git 屬於分散式的版本控制系統，所以開發可以在離線 local 環境下開發，等到有網路時再將自己的程式push 到 Remote 環境或 pull 下其他開發程式碼進行整合。在 Local 中我們又分為 working directory（工作資料夾）、staging area（暫存區）和 repositories（檔案庫）。



Git下載/安裝

  • 下載git:

    另外 Github 當作遠端托管程式的環境，所以我們也可以根據作業系統安裝 當作操作工具（內建安裝Git）。

    GIT下載點
    Github桌面版
  • 安裝完成:

    安裝完成後，打開終端機（terminal）/CMD(command line) 輸入下方指令， 確認是否安裝成功或確認版本 

    $ git --version
  • 設定使用:

    設定你的帳戶，讓 Git 知道這台電腦做的修改要連結到哪一個使用（Name、Email 的格子是填入自己的github中的名字(username)跟信箱）：

    $ git config --global user.name "Name"

    $ git config --global user.email "Email"

    做完後，前置設定已經設定完成，剩下可以使用clone後在使用pull跟push的動作。


GITHUB上傳

移動至 https://github.com/ 後，在Repositories右方有個NEW 創建倉庫，Repository name 隨意輸入名稱後，勾選 Initialize this repository with a README會出現README.mdmd的檔案。先把repository 抓到在桌面上。

名稱 指令 內容
clone git clone 'url' #'url''把github上面的urlurl打上去，無淤加任何 " "、' '
status git status #作為檢查資料是否又做更改、更新的動作。
add git add * #把所有的檔案抓到 load 儲存。
commit git commit -m "info." #輸入檔案訊息(標註)
push git push #推上github上面
